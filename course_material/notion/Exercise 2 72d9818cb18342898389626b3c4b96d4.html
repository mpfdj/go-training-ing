Exercise 2

Objective

Develop a Go program that parses vCards to extract contact information. Rather than immediately displaying the parsed data, first store the contact details within appropriate Go data structures. This step ensures organized and structured handling of contact information, allowing for more efficient and effective display and further manipulation.

Summary

  1. Review vCard format to identify target fields.
  1. Define a struct in Go to hold contact information.
  1. Parse vCards and populate the struct.
  1. Loop over and display the stored contact information.

vCard specification

The vCard data format can be used to describe information about individuals and is defined in RFC6350, although we will use a slightly simplified version. The information is displayed line-by-line where each line follows the following format: KEY:VALUE1[,VALUE2]. A single vCard file can contain multiple contacts, and each of the following components can appear once per contact:

  • FN; Full name
  • TEL; Telephone number
  • EMAIL; Email address

Example

BEGIN:VCARD
FN:John Doe
TEL:1234567890
EMAIL:john.doe@example.com
END:VCARD

Hints

  • Parsing flow

    You can create a variable which holds the contact data which will be updated in a for loop. Whenever you encounter an END:VCARD tag you can add it to a slice and repeat the process.

Follow up

Extend the program and add support for the N component. The format of this component is defined in the RFC:

The structured property value corresponds, in sequence, to the Family Names (also known as surnames), Given Names, Additional Names, Honorific Prefixes, and Honorific Suffixes. The text components are separated by the SEMICOLON character (U+003B).

Input data

BEGIN:VCARD
N:Doe;John;Peter;Dr.
TEL:1234567890
EMAIL:john.doe@example.com
END:VCARD
