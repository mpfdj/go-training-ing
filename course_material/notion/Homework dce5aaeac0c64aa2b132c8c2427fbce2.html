Homework

Objective

Trainees will build a simple Markdown to HTML converter using Go, to enhance their understanding of Go's file handling, text processing capabilities, and to get hands-on experience with practical project development in Go. The final version will recurse through all files or a directory convert each markdown file to HTML, concurrently, and store it in the same directory.

Summary

  1. Set up a new Go project directory.
  1. Create a function to read a Markdown file.
  1. Parse the Markdown content to identify headers, paragraphs, and lists.
  1. Create functions to convert identified Markdown elements to HTML.
  1. Write the converted HTML content to a new file.
  1. Include error handling to manage potential read/write or parsing errors.
  1. Test the converter with various Markdown inputs.
  1. Implement recursion and concurrency.

Specification

We will use the CommonMark specification as a reference throughout this exercise. Don’t focus too much on the details. Things like automatic HTML encoding, backslashes, insecure characters etc. are not required. We’re not aiming for 100% compatibility as long as simple input results in the expected output.

Example

Converting a single document

Given the following input:

# Heading
Paragraph text.

- List item 1
- List item 2

The program should create the following output:

<html>
<head></head>
<body>
<h1>Heading</h1>
<p>Paragraph text.</p>

<ul>
  <li>List item 1</li>
  <li>List item 2</li>
</ul>
</body>
</html>

Converting multiple documents

With the following files:

/files
  /a
    /b
      test.md
      example.md
  /c
    hello.md

Running your program will result in the following output:

/files
  /a
    /b
      test.md
      test.html
      example.md
      example.html
  /c
    hello.md
    hello.html
